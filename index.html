How to replace a common cloud computing service with a self-hosted cloud

Introduction



What is Cloud Computing?

Cloud Computing is delivering computing power a service over a network rather than having physically having the computing resources at the customer. It is also a general term for anything that involves delivering hosted services over the Internet. These services are broadly divided into three categories: Infrastructure-as-a-Service (IaaS), Platform-as-a-Service (PaaS) and Software-as-a-Service (SaaS). There are two types of cloud which are public and private. A public cloud sells services to anyone on the internet while a private cloud is a data center that supplies hosted services to a limited number of users. Some examples of cloud computing services are Amazon Web Services(AWS), Google Cloud Platform, Microsoft Azure and many more. The main purpose of cloud computing is to provide easy and scalable access to computing resources and IT services.

Why the word 'Cloud'?

The word Cloud was inspired by the cloud symbol that's often used to represent the Internet, such as the location of various network devices in flowcharts and diagrams.

Why use cloud computing?

There are a few benefits from using cloud computing:

     1. Lower Cost - no need for physical hardware
     2. Available 24/7 - as long as there's internet
     3. Easily Managable - everything in one cloud
     4. Accessable Anywhere - access from any device anywhere
     5. Automatic Updates - less workload for the team
     6. Sustainable - less physical hardware reduces carbon footprints

What is a self-hosted cloud?

Self-hosted cloud is a cloud hosted through your own server where we have full control over our own data and internet is needed to access the data. How much storage space also depends on the owner of the server.

Instructions


Step 1 - Install NextCloud on Ubuntu


Run installation for NextCloud on Ubuntu 18.04 (a linux based Operating system)

Step 2 - Install Web Server (Apache2)


We will be using Apache2 as our web server


Code

sudo apt install apache2

sudo sed -i "s/Options IndexesFollowSymLinks/Options FollowSymLinks/"/etc/apache2/apache2.conf

sudo systemctl stop apache2.service

sudo systemctl start apache2.service

sudo systemctl enable apache2.service

Step 3 - Install Database and PHP


We'll use MariaDB for our database.


Code

sudo apt install mariadb-server mariadb-client

sudo systemctl stop mariadb.service sudo systemctl start mariadb.service sudo systemctl enable mariadb.service - sudo

Step 4 - Create NextCloud Database


Setup a database for our cloud server.


Code

sudo mysql -u root -p

CREATE DATABASE nextcloud;

CREATE USER 'nextclouduser'@'localhost' IDENTIFIED BY 'password_here';

GRANT ALL ON nextcloud.* TO 'nextclouduser'@'localhost' IDENTIFIED BY 'user_password_here' WITH GRANT OPTION;

FLUSH PRIVILEGES;

EXIT;

Step 5 - Configure Firewall


Secure server with Secure Socket Layer(SSL)


Code

sudo nextcloud.enable-https self-signed

sudo ufw allow 80,443/tcp

Step 6 - Login NextCloud Server on Browser


1. Open browser and type in your IP address

2. Choose 'Advance' and 'Add Exception'

3. Login using system username and password

Step 7 - Allow Other Devices


Through port forwarding to channel data to specific devices

1. [your IP]/firewall.htm

2. Create new port forwarding rule(which points to cloud server)
